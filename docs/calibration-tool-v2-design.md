# 新版标定工具设计

日期：2026-07-17  
状态：已确认，待原型与实现

## 1. 目标与边界

新版标定工具运行在配置人员的电脑上，负责生成和维护增氧机预置点标定配置。它不承担水花识别，也不改变识别算法。

标定工具的产物分为两类：

- 部署配置：供管理系统下发到识别代理，只包含识别运行所需的当前有效配置。
- 标定归档：供人工复核、维护和审计，包含配置版本、原始快照和标注快照。

完整标定仅支持电脑端，目标最小分辨率为 `1280 x 800`；不承诺手机或平板的完整 ROI 编辑能力。

## 2. 页面结构

采用单页工业工作台，而不是强制逐页向导。配置人员可在任意时刻重新移动云台、切换预置点和重新取景。

- 顶部横向连接栏：设备 ID、通道、流类型、连接状态、加载视频。
- 左侧操作栏：预置点列表、覆盖状态、云台微调、新建或转到预置点。
- 中央主工作区：视频预览、判稳状态、冻结帧和双 ROI 画布。
- 右侧当前标定卡：完成检查单、目标信息、保存、更新、导出和新建下一个标定。
- 侧边抽屉：已有标定、历史版本和恢复操作。
- 底部抽屉：诊断信息、播放器事件和后端日志。

宽屏使用三栏；`1024-1439px` 使用两栏并将右侧保存区移到下方；低于 `1024px` 显示应使用电脑端的提示。

## 3. 标定流程

```text
连接设备
-> 选择已有预置点，或云台取景后保存新预置点
-> 转到预置点
-> 机械稳定、视频追帧、视觉判稳
-> 冻结画面
-> 标定识别 ROI 与对焦锚点 ROI
-> 填写目标信息
-> 保存或更新标定
-> 保留当前结果，或新建下一个标定
```

每条标定必须绑定一个已存在的预置点。云台移动或预置点切换后，冻结帧和两个 ROI 立即失效，必须重新判稳、冻结和标定。

保存成功后不自动清空页面。用户可查看结果或显式点击“新建下一个标定”；该动作保留设备会话、视频流、预置点列表和云台控制，清除目标信息、冻结帧与 ROI。

## 4. 双 ROI 交互

冻结帧上始终显示两个 ROI，但一次只编辑一个：

- 识别 ROI：红色，用于框定水花主要出现区域。
- 对焦锚点 ROI：蓝色，用于框定稳定、清晰、有纹理的固定参照物。

通过分段切换选择当前编辑对象。当前 ROI 使用实线、控制点和高对比度；另一个 ROI 使用半透明虚线且锁定。重新绘制和清除只作用于当前 ROI，切换编辑对象不会清除另一个 ROI。

## 5. 状态与保护

主界面只显示配置人员可理解的状态：机械稳定、同步视频、检查画面稳定性、可冻结、视频异常。技术指标和日志进入诊断抽屉。

右侧“当前标定”检查单依次确认：

1. 设备已连接。
2. 已绑定预置点。
3. 已冻结稳定画面。
4. 识别 ROI 已完成。
5. 对焦锚点 ROI 已完成。
6. 目标编号与名称已填写。

检查单状态包括未完成、进行中、已完成、需重新完成、异常和已保存。只有全部完成时才可保存或更新标定。

存在未保存改动时，切换设备、预置点、加载其他标定或新建下一条标定必须确认丢弃草稿。

## 6. 预置点覆盖与已有标定

预置点列表显示：已标定、未标定、当前标定中、不纳入巡检。覆盖概览只统计应纳入巡检的预置点。

选择已有标定时，页面显示目标名称、最后更新时间和标定状态；支持加载查看或重新标定。更新已有配置必须明确确认覆盖。

## 7. 双快照与版本历史

每个新版本保存两种快照：

- 原始快照：未经标注的冻结画面，保留原始视觉证据。
- 标注快照：在原始画面上绘制红色识别 ROI、蓝色对焦锚点 ROI，并显示目标、设备、通道、预置点、时间和 ROI 坐标。

历史版本不可变。当前生效配置与历史版本分离：

```text
data/calibrations/<device>_<preset>.json
data/calibration_history/<device>_<preset>/v0001/
data/calibration_history/<device>_<preset>/v0002/
```

历史版本包含完整配置、原始快照和标注快照。恢复历史版本时不直接复用旧版本号，而是以恢复内容创建一个新版本，并记录 `restoredFromVersion`。

旧标定原样兼容：未带版本或标注快照的记录显示为旧版标定。首次更新时，旧记录归档为 `v0001`（`legacy=true`），新的保存结果写入 `v0002`。本机不自动清理版本历史。

## 8. 导出规则

- 导出当前配置：仅导出一个当前有效部署 JSON。
- 导出全部配置：仅导出全部当前有效部署 JSON，适用于首次部署。
- 标定归档导出：导出 ZIP，包含当前/历史配置、原始快照、标注快照和相对路径清单。

部署配置不要求携带快照。识别代理继续消费当前有效配置中的设备、通道、预置点、`roi` 和 `focusAnchorRoi`；快照只服务于标定复核与归档。

## 9. 视觉语言

采用浅色、克制的工业工作台：暖白/浅灰背景，深色文字，深青绿色作为普通主操作色。红色仅用于识别 ROI、危险和覆盖确认；蓝色仅用于对焦锚点 ROI 及其说明；琥珀色用于等待与注意；绿色用于成功、已连接和可冻结。

状态不只依赖颜色，必须同时使用图标和文字。动画仅用于进行中和短暂成功提示，不使用干扰操作的装饰动画。

## 10. 非目标

- 不在本轮接入管理系统上传接口或定义其传输协议。
- 不在本轮改变识别算法或重新引入构图守卫。
- 不在本轮实现移动端完整标定。
- 不自动删除历史标定或快照。
