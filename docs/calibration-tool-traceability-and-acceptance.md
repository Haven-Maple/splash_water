# 标定工具工作留痕与验收规范

## 1. 文档目的

本文档用于约束标定工具开发过程中的工作留痕方式、阶段提交要求、问题记录要求和最终验收口径。

目标不是增加流程负担，而是保证：

- 后续实现可复盘
- 问题排查有证据
- 交接给其他对话或开发者后不丢上下文
- 开发完成后可以快速验收

## 2. 适用范围

本文档适用于第一阶段标定工具的实现过程，包括：

- 后端接口封装
- 前端页面开发
- 本地配置保存
- 流预览联调
- PTZ 联调
- 预置点联调
- ROI 框选功能

## 3. 必须保留的工作留痕

第一阶段实现过程中，至少需要保留以下 4 类痕迹。

### 3.1 代码变更留痕

要求：

- 每完成一个相对独立的功能点，就记录本次改动内容
- 记录应能说明“改了什么”和“为什么改”

建议方式：

- 在开发对话中同步说明
- 或写入开发日志文档

建议记录内容：

- 功能名称
- 涉及文件
- 关键设计决策
- 未解决问题

### 3.2 联调留痕

要求：

- 每次对接大华接口时记录调用结果

至少记录：

- 调用时间
- 接口名称
- 输入参数摘要
- 返回码
- 返回核心字段
- 是否成功
- 失败原因

注意：

- 不记录明文密钥
- 不记录完整 token

### 3.3 标定结果留痕

每次完成一次成功标定时，必须至少保留：

- 标定配置 JSON
- 标定时冻结帧截图
- ROI 坐标
- 关联设备 ID 和预置点 ID

### 3.4 问题处理留痕

每遇到一个非瞬时问题，都应记录：

- 问题现象
- 复现方式
- 初步猜测
- 最终原因
- 解决办法
- 是否需要长期规避

## 5. 实现过程中建议的里程碑提交点

为了方便交接和验收，建议按以下节点提交阶段成果。

### 5.1 里程碑一：后端基础可用

完成标准：

- 能获取 token
- 能调 `deviceOnline`
- 能返回统一结果

应提交：

- 代码
- 接口调用日志
- 简单接口说明

### 5.2 里程碑二：流获取与 PTZ 可用

完成标准：

- 能获取 FLV 或 HLS
- 能成功发 PTZ 控制命令

应提交：

- 联调记录
- 成功调用截图或日志
- 已知延迟问题记录

### 5.3 里程碑三：预置点能力可用

完成标准：

- 能查询预置点
- 能保存预置点
- 能跳转预置点

应提交：

- 成功联调日志
- 预置点保存/回查结果

### 5.4 里程碑四：前端标定页面可用

完成标准：

- 能预览流
- 能点击 PTZ 调整
- 能查询/保存预置点

应提交：

- 页面录屏
- 当前未解决问题

### 5.5 里程碑五：完整标定闭环成立

完成标准：

- 能冻结帧
- 能框选 ROI
- 能保存配置 JSON
- 能保存截图

应提交：

- 一份真实配置文件
- 一张真实冻结帧截图
- 一次完整流程录屏或日志

## 6. 最终验收输入要求

当实现完成并交回验收时，至少应提供以下内容：

- 代码目录
- 运行方式说明
- 一份示例配置文件
- 一份接口联调结果摘要
- 一次完整标定操作说明
- 已知问题列表

如果缺少这些内容，验收效率会明显下降。

## 7. 最终验收检查表

验收时建议按以下清单逐项检查。

### 7.1 后端检查

- 是否能成功获取 `AppAccessToken`
- 是否能成功查询设备在线状态
- 是否能成功获取 FLV 地址
- 是否能成功获取 HLS 地址
- 是否能成功控制 PTZ
- 是否能成功查询预置点
- 是否能成功保存预置点
- 是否能成功跳转预置点
- 是否能成功保存标定配置

### 7.2 前端检查

- 是否能输入设备参数
- 是否能显示在线状态
- 是否能正常播放预览流
- 是否能进行 PTZ 微调
- 是否能查询和选择预置点
- 是否能冻结当前帧
- 是否能框选 ROI
- 是否能填写目标信息并保存

### 7.3 留痕检查

- 是否有本地日志
- 是否有标定配置 JSON
- 是否有冻结帧截图
- 是否有问题记录文档
- 是否有联调记录

## 8. 验收通过标准

第一阶段标定工具验收通过的最低标准是：

- 可以连接真实设备
- 可以查看预览画面
- 可以通过 PTZ 调整画面
- 可以保存并调用预置点
- 可以冻结画面并框选 ROI
- 可以保存本地标定配置
- 可以保留完整留痕

不要求：

- 私有流地址打通
- 多设备批量管理
- 夜间专项能力
- 与识别程序完整联动

## 9. 常见验收失败原因

以下情况容易导致验收不通过：

- 只能演示页面，无法连真实设备
- 只能看流，不能真正控制 PTZ
- 保存预置点后无法回查
- ROI 只能临时画，不能落配置
- 配置保存了，但没有截图证据
- 没有日志，出了问题无法复盘

## 10. 建议交接方式

把实现任务交给另一个对话时，建议直接附上以下材料：

- `docs/calibration-tool-design.md`
- `docs/calibration-tool-implementation-plan.md`
- `docs/calibration-tool-traceability-and-acceptance.md`

建议交接语句：

“请按这三份文档实现第一阶段标定工具。实现时优先保证真实设备可联调和留痕完整，不要先做大而全页面。每完成一个里程碑，请同步维护开发日志、问题清单和联调记录，最终交付时附运行方式、示例配置和已知问题。”
