# 2026-07-19 发布基线

## 发布范围

本次基线同时固化标定工具 V2 与夜间识别运行时加固。识别程序仍独立于标定网页运行：网页产出标定配置与快照，识别侧消费导出的配置，不依赖前端运行时。

### 标定工具 V2

- 中文桌面工作台，顶部横向完成设备、通道与视频流连接。
- 支持 splash `roi`（红色）和 `focusAnchorRoi`（蓝色）双 ROI 标定；前者用于识别，后者用于画面判稳与采样质量保护。
- 冻结前必须完成对应预置点的转点与视觉判稳；设备、通道、预置点或云台移动后，冻结状态会失效。
- 保存时同时保留原始快照与带 ROI 标注的复核快照，生成不可变 `v000N` 历史版本；恢复历史版本会创建新的当前版本，不会改写历史。
- 支持导出全部部署配置、当前配置，以及包含标定档案和快照的归档包。

### 夜间识别运行时

- 夜间运行前提是摄像头固定 IR 黑白成像；不支持识别过程中彩色与 IR 自动切换。
- 保留 startup freshness、scene-mode stability、visual readiness、sample-quality guard、temporal voting 与 replay 证据链。
- `night_ir` 启用 ROI 容忍：基础 ROI 不能判为 `has_splash` 时，对 `+-8%` 平移及 `1.1x` 扩展形成的 18 个候选做低成本预筛，最多对 3 个候选运行完整门控与时序投票。
- 基础 ROI 已通过时不进行多候选完整检测，避免为正常轮次增加不必要计算。
- FLV 流打开超时为 `6000 ms`，单次读帧超时为 `3000 ms`；识别前阶段发生明确流异常时会重开一次 session，并将真实读流耗时、失败次数与恢复结果写入 replay 和伪多点结果。
- 伪多点默认 `round-timeout-policy=warn_only`：正确识别但超过时延目标记为 `recognized_but_over_slo`，不改判业务识别结果；显式使用 `fail` 才会作为严格超时失败。

## 已有证据与验证边界

- 既有固定 IR、单点标定条件下，已完成连续 100 轮夜间现场测试无失败；该结论不外推到新预置点、多目标轮询、不同镜头、天气变化或运行中成像模式切换。
- ROI 容忍离线回放中，2026-07-17 `18:56` 的有水花样本为 `10/10 has_splash`，其中 9 轮由候选 ROI 救回；同期无水花样本为 `10/10 no_splash`。
- 自动化回归覆盖标定历史/导出、ROI 容忍、流超时与重开、伪多点时延语义；本次发布前应运行完整的 Python 单测和编译检查。

## 未解决事项与下一步

大华云端 `turnPreset` 控制请求仍共用 `20 s` HTTP 读取超时。现场已出现 `open.cloud-dahua.com:443` 读取超时，失败发生在转预置点阶段，尚未进入识别。这是控制面网络超时，不是 HTTP 443 状态码，也不应归因于 ROI 或 splash 阈值。

下一项修改应独立处理该问题：为 `turnPreset` 设定较短的连接/读取超时，只对无响应网络超时重试同一预置点一次，并输出尝试次数、每次耗时和可区分的失败原因。失败后不得继续执行识别。不要在同一修改中调整 ROI 候选、夜间硬门控或时序阈值。

## 明确不做

- `night_visible` 第三套夜间全彩识别 profile
- 识别过程中彩色/IR 切换处理
- 基于单张标定参考图的构图守卫
- HLS 自动降级
- 每个预置点独立维护 splash 阈值
