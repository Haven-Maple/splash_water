# 标定工具实现方案

## 1. 文档目的

本文档用于明确第一阶段标定工具的实现目标、页面功能、后端接口适配方式、配置数据结构以及完整操作时序，作为后续开发的直接依据。

本工具仅服务于第一阶段单点验证，不以完整运维平台或多设备管理平台为目标。

## 2. 工具目标

标定工具的目标是帮助开发或运维人员完成以下工作：

- 连接指定摄像头
- 查看当前摄像头预览画面
- 手动控制 PTZ 将目标增氧机调整到合适位置
- 创建或覆盖保存一个预置点
- 在当前画面上框选识别区域
- 保存单点标定配置，供后续识别程序直接使用

换句话说，本工具解决的是：

`设备 -> 预置点 -> 画面 -> ROI -> 配置`

这一段链路。

## 3. 工具边界

### 3.1 第一阶段必须支持

- 单设备操作
- 单点标定
- 设备在线检查
- FLV 预览
- HLS 备用预览
- PTZ 方向与变倍控制
- 查询已有预置点
- 创建或覆盖保存预置点
- 在画面上框选 ROI
- 保存本地配置

### 3.2 第一阶段可以不做

- 多设备批量管理
- 多用户权限
- 云端配置同步
- 历史版本回滚
- 自动识别目标位置
- 与识别程序深度联动
- 对私有 RTSP 拉流地址的兼容

## 4. 接口清单与用途

当前已确认可用接口如下。

### 4.1 鉴权接口

- `POST /open-api/api-base/auth/getAppAccessToken`

用途：

- 获取后续所有开放接口访问所需的 `AppAccessToken`

### 4.2 设备在线状态接口

- `POST /open-api/api-iot/device/deviceOnline`

用途：

- 在进入标定前校验设备是否在线
- 失败时给出明确提示，避免后续 PTZ 或预览请求无意义重试

### 4.3 预置点查询接口

- `POST /open-api/api-aiot/device/queryPreset`

用途：

- 获取当前设备已有预置点
- 用于选择已有预置点进行覆盖或跳转

### 4.4 预置点跳转接口

- `POST /open-api/api-aiot/device/turnPreset`

用途：

- 跳转到某个已有预置点
- 便于复查、重标定和配置验证

### 4.5 预置点配置接口

- `POST /open-api/api-aiot/device/configPreset`

用途：

- 创建新预置点
- 覆盖或更新已有预置点

### 4.6 PTZ 控制接口

- `POST /api-aiot/device/controlMovePTZ`

用途：

- 控制云台上下左右、斜向移动和变倍

已知关键参数：

- 左右/上下移速：范围 `(-1, 1)`
- 移动持续时间：单位毫秒

实现建议：

- 支持点按微调
- 支持短时连续移动
- 不建议第一版做长时自动巡航

### 4.7 HLS 预览流接口

- `POST /open-api/api-iot/device/getHlsLiveList`

用途：

- 获取 HLS 预览地址
- 作为 FLV 不可用时的降级方案

### 4.8 FLV 预览流接口

- `POST /open-api/api-iot/device/queryDeviceFlvLive`

用途：

- 获取低延迟预览流地址
- 作为标定工具默认预览方案

### 4.9 私有拉流接口

- `POST /open-api/api-aiot/device/getStreamUrl`

用途：

- 作为后续研究项保留

当前判断：

- 不作为第一阶段标定工具的核心依赖
- 因为返回的大华私有地址目前无法通过 VLC 直接使用

## 5. 预览流策略

### 5.1 推荐优先级

标定工具建议按以下顺序选择预览流：

1. FLV
2. HLS
3. 私有流地址暂不接入

### 5.2 原因

- FLV 延迟通常低于 HLS，更适合 PTZ 手动调节
- HLS 可作为兼容降级方案
- 私有流地址当前可用性未验证，不能作为第一阶段阻塞依赖

### 5.3 风险提醒

即使使用 FLV，仍可能存在播放器缓冲导致的轻微延迟。

因此工具内建议增加以下能力：

- 刷新当前帧
- 冻结当前帧后再框选 ROI
- 显示当前预览流类型

## 6. 页面功能清单

第一版建议做成单页面工具，页面分为 4 个区域。

### 6.1 设备连接区

功能：

- 输入或选择设备 ID
- 输入通道信息
- 触发在线状态检查
- 获取预览流

显示信息：

- 设备在线/离线状态
- 当前预览流类型
- 当前预置点 ID

### 6.2 视频预览区

功能：

- 播放 FLV 预览流
- FLV 不可用时切换到 HLS
- 刷新当前预览
- 截取当前帧
- 冻结当前帧
- 在冻结画面上框选 ROI

显示信息：

- 当前帧截图
- ROI 坐标
- 画面时间或截图时间

### 6.3 PTZ 控制区

功能：

- 上、下、左、右、左上、右上、左下、右下移动
- 放大、缩小
- 微调步长选择
- 持续时间输入或档位选择

第一版建议交互：

- 单击按钮发送一次短动作
- 不做复杂摇杆

### 6.4 预置点与配置区

功能：

- 查询已有预置点
- 输入新预置点 ID 或选择已有预置点
- 保存当前 PTZ 位置为预置点
- 跳转到指定预置点
- 输入目标名称
- 保存完整标定配置

建议显示：

- 当前配置摘要
- 最近一次保存结果

## 7. 后端适配模块清单

建议后端拆分为以下 5 个适配模块。

### 7.1 鉴权模块

职责：

- 获取 `AppAccessToken`
- 缓存 token
- 统一签名和请求头构造

说明：

- 现有 `request_sign_utils.py` 已经是较好的起点
- 但建议后续整理成可复用客户端，而不是单文件示例脚本

### 7.2 设备状态模块

职责：

- 查询设备在线状态
- 输出标准化设备状态结果

### 7.3 预览流模块

职责：

- 获取 FLV 地址
- 获取 HLS 地址
- 统一返回给前端播放

建议返回结构：

- `stream_type`
- `stream_url`
- `expires_at` 或有效期信息
- `fallback_available`

### 7.4 PTZ 与预置点模块

职责：

- 调用 PTZ 控制接口
- 查询预置点
- 保存预置点
- 跳转预置点

建议统一封装：

- `move_ptz`
- `query_presets`
- `save_preset`
- `turn_preset`

### 7.5 标定配置模块

职责：

- 保存本地标定配置
- 读取已有配置
- 校验 ROI 坐标和必要字段

## 8. 配置文件结构建议

第一阶段建议用本地 JSON 保存标定结果。

建议结构如下：

```json
{
  "version": "1.0",
  "deviceId": "DEVICE_ID",
  "channelId": "CHANNEL_ID",
  "targetId": "AERATOR_001",
  "targetName": "1号增氧机",
  "presetId": "PRESET_001",
  "presetName": "1号增氧机预置点",
  "roi": {
    "x": 320,
    "y": 180,
    "width": 260,
    "height": 220
  },
  "streamPreference": "flv",
  "captureMode": "snapshot",
  "captureDurationMs": 1500,
  "ptzSettleMs": 1200,
  "notes": "",
  "updatedAt": "2026-07-01T12:00:00+08:00"
}
```

建议字段说明：

- `deviceId`：设备唯一标识
- `channelId`：通道标识
- `targetId`：目标增氧机内部编号
- `presetId`：预置点编号
- `roi`：识别区域
- `streamPreference`：首选预览流类型
- `captureMode`：后续识别采集方式
- `captureDurationMs`：后续识别默认采集时长
- `ptzSettleMs`：转到预置点后的稳定等待时间

## 9. 一次完整标定流程

建议完整标定按以下时序执行：

1. 输入设备 ID 和必要参数
2. 调用 `deviceOnline` 检查设备在线状态
3. 获取 FLV 预览流
4. 前端开始播放预览
5. 用户通过 `controlMovePTZ` 微调摄像头位置
6. 用户确认目标增氧机进入合适视野
7. 用户输入预置点 ID 或选择已有预置点
8. 调用 `configPreset` 保存当前 PTZ 位置
9. 用户冻结当前帧
10. 用户在冻结画面上框选 ROI
11. 用户填写目标名称等信息
12. 保存本地标定配置
13. 可选调用 `turnPreset` 复查保存结果

## 10. 关键交互细节建议

### 10.1 PTZ 微调

由于 PTZ 接口使用“速度 + 持续时间”模型，第一版建议预设 3 档微调：

- 小步：低速 + 150ms
- 中步：中速 + 300ms
- 大步：中高速 + 500ms

这样用户无需每次手输参数。

### 10.2 ROI 框选

建议 ROI 框选只在冻结帧上进行，不在实时视频上直接画框。

原因：

- 交互更稳定
- 更容易避免流延迟干扰
- 更容易保存精确坐标

### 10.3 配置保存

建议保存时同时落两类信息：

- 标定配置 JSON
- 对应冻结帧截图

这样后续调试识别程序时可以直接回看标定时的参考画面。

## 11. 最小后端接口建议

如果我们自己做一层本地服务，建议先暴露这些接口给前端：

- `POST /api/auth/token`
- `POST /api/device/online`
- `POST /api/stream/flv`
- `POST /api/stream/hls`
- `POST /api/ptz/move`
- `POST /api/preset/query`
- `POST /api/preset/save`
- `POST /api/preset/turn`
- `POST /api/calibration/save`
- `GET /api/calibration/get`

说明：

- 这些是我们工具自己的本地接口，不是大华开放平台原始接口
- 这样可以把签名、token 和厂商细节封装在后端

## 12. 风险与处理建议

### 12.1 FLV 仍有延迟

处理建议：

- 增加冻结帧操作
- 用短步 PTZ 微调代替持续移动

### 12.2 预置点保存成功但画面略偏

处理建议：

- 保存后立即 `turnPreset` 回查
- 必要时再次覆盖保存

### 12.3 ROI 过小导致后续识别不稳定

处理建议：

- 第一版 ROI 适当留安全边界
- 不要求只框极小区域

### 12.4 配置结构后续演进困难

处理建议：

- 从第一版开始保留 `version` 字段
- 预留备注字段和采集参数字段

## 13. 当前推荐结论

当前标定工具完全可以按以下方案开工：

- 默认预览流使用 FLV
- HLS 作为降级方案
- 私有 `getStreamUrl` 暂不纳入第一阶段主链路
- 页面先做成单页单设备标定工具
- 后端先封装鉴权、预览、PTZ、预置点和配置保存 5 类能力
- 标定结果先保存为本地 JSON + 截图

## 14. 下一步开发建议

建议按以下顺序进入实现：

1. 先定义本地配置结构和工具后端接口
2. 封装大华鉴权与基础请求客户端
3. 打通 FLV 获取与预览
4. 打通 PTZ 控制与预置点保存
5. 实现冻结帧 + ROI 框选 + 配置保存
6. 最后补 HLS 降级逻辑
