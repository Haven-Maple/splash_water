# 2026-07-20 发布基线

## 本次范围

本次发布加固大华控制面与 FLV 数据面故障处理，不调整 splash ROI、夜间硬门控、时序投票或 sample-quality 阈值。

### 预置点控制

- `turnPreset` 使用独立的 `2 s` 连接超时和 `3 s` 读取超时。
- 仅对无 HTTP 响应的网络连接、读取超时或连接中断重试一次，退避 `500 ms`。
- 每次网络重试重新生成请求签名、`Nonce` 和 trace ID；明确的 HTTP 或业务拒绝不会重试。
- token 首次获取、缓存失效刷新及 401 刷新均继承控制面短超时；401 刷新不计入网络无响应重试。
- 伪多点结果记录预置点调用尝试、耗时、失败类别、trace ID 与 token 获取耗时。

### FLV 数据面

- 初始 FLV 打开上限为 `6000 ms`，单次读帧上限为 `3000 ms`。
- 发生读流异常后使用跨 startup、readiness、sample-quality 阶段共享的恢复预算：最多重开 `2` 次，退避 `500 ms -> 1000 ms`，总预算 `10000 ms`。
- 重开时重新获取 FLV 地址并创建新的 FFmpeg capture；流 URL 请求、token 获取、401 刷新和 FFmpeg 打开均受同一绝对 deadline 约束。
- 重开 session 已打开与实际读到新帧分别记录为 `sessionOpened` 和 `streamRecovered`，不会把打开 capture 误记为流已恢复。
- 预算耗尽返回 `stream_recovery_exhausted`，不再把持续断流伪装成场景切换、readiness 或无水花判断。

## 当前策略

- 固定 IR 黑白成像是夜间运行前提；不引入 `night_visible` 或运行中彩色/IR 切换。
- 遇到 `stream_recovery_exhausted` 时，将其作为摄像头到 FLV 网关的可观测运行故障处理，不调整 splash 识别阈值。
- `round-timeout-policy=warn_only` 下，正确识别但超出时延目标只记为 SLO 警告；识别执行失败仍是失败轮次。

## 下一步

等待多增氧机环境后，验证多预置点轮询、不同目标标定及配置下发边界。现场继续保留控制面和流恢复诊断，但在新证据出现前不扩大重连次数、不引入 HLS 自动降级，也不调整夜间识别阈值。
